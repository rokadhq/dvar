# Security Policy

Dvar is security-sensitive infrastructure. Please do not open public issues for suspected vulnerabilities.

Report vulnerabilities privately through GitHub Security Advisories for `rokadhq/dvar`. Include the affected version, reproduction steps, impact, and any suggested mitigation. Do not include live credentials, private customer data, or destructive proof-of-concept payloads.

## Supported versions

Until Dvar reaches 1.0, security fixes are provided for the latest published prerelease only. Pre-1.0 APIs and policy fields may change with migration guidance.

## Scope statement

Dvar protects actions only when they pass through a Dvar interception boundary. A policy bypass caused by direct tool invocation outside Dvar is an integration failure, not an enforcement guarantee.
